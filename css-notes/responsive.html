Media Queries

Responsive design in CSS refers to the ability to change the layout based on the size of the screen, so the same html can be used for desktop, tablet, and mobile.

The main tool in the arsenal of responsive design is the @media query.

Where to Put Media Queries

Put media queries at the end of your CSS file. They override other styles, and as we know, the last styles in the cascading style sheet get preference, so your media queries will fail if they come before the styles they are intended to override.

Breakpoints

Breakpoints are cutoffs for different viewing devices. The following are the instructor's, Josh Comeau's, recommended breakpoints:

0-550px — Mobile
550-1100px — Tablet
1100-1500px — Laptop
1500+px — Desktop

The modern iphone has around 375px, but some androids have 400px, so it's best to give a buffer area to allow for differences in mobile devices, etc.

While these recommended breakpoints are in pixels, it's really best to use rem for breakpoints. For example, if a user increases their default font size, it might be better to have the tablet layout on desktop in those cases, because rem will scale with the default font size.

The All Important Order of Media Query Breakpoints

So great, you've got some breakpoints figured out, and if you write an exclusive media query that only allows for the styles within to work if the screen is bigger than 550px AND if the screen is smaller than 1100px, then go ahead and slap that baby down in any order...

Clamping

Before we explain clamping, let's talk about how min-width and max-width can help with responsive design.

This box has a min-width of 500px and a max-width of 700px, as well as a width of 80%. When all of these values are set, it allows this box to respond to the amount of space it has been given. Width: 80% is its ideal width, but it is not allowed to grow smaller than 500px nor larger than 900px. These constraints can be quite handy when creating responsive webpages, because you won't want the box to remain at 80% on mobile, so the min-width will keep the box wider as the page shrinks. However! If the screen goes smaller than 500px, it will create a scrollbar and break the layout! In fact, it was breaking my layout. Typography on Mobile

Typography on mobile and desktop should be the same for the most part. For basic body text, the size should be at least 16px (1rem). (Remember to use rem for font-size).

Default Sizes

The problem on mobile is that some html elements have a default size smaller than 16px. Textarea and input default to a smaller size, yet iOS will automatically zoom in on text input fields with font-sizes smaller than 16px. Headings

Headings on mobile, however, might need to be adjusted, because large, long headings can take up the entire screen.

One way is to use media queries and make the headings a more appropriate size.

Another way is to use fluid typography. Fluid typography can be achieved by using vw, or viewing width. However, you'll want to use clamp as well, because a heading of 6vw will become VERY large on wide screens and unreadably tiny on small screens, so it's important to set mininum and maximum sizes. Something like: font-size: clamp(1.5rem, 6vw, 3rem);

But...there's a problem. Using a fluid unit like vw disables zoom. That means if the user zooms in to make the font bigger, it won't work for the headings. To fix this, mix units on the ideal size in the clamp property: font-size: clamp(1.5rem, 4vw + 1rem, 3rem); This will make the text zoomable again.
