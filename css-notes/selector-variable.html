SELECTORS VARIABLES

Selectors

The main CSS selectors are element, id, and class. Id and class have a higher specificity than an element. There are many other ways to combine element, classes, and pseudo-classes to target specific items. For a full list of combinators and pseudo-classes, check out W3schools reference list.

Combinators

.ancestor .descendant Use a space to target any descendants of an element, id, or class.

.ancestor.descendant No space between classes? That means both classes must exist on the same element.

.mama > .child Here the child must be a direct descendant of the parent.

.sister ~ .brother Will choose the siblings of .sister who have a class of .brother. They don't have to be directly adjacent.

.sister + .brother Siblings again. This time, they DO have to be directly adjacent, so that .brother follows immediately after .sister.

.parents, .children Party time. Give me all the parents and all the children.

Pseudo Classes

Pseudo classes are super useful. They allow you to pick just the first child of a class or element, or only set a style on mouse hover.

.flashy-icon:hover Set styles for the flashy-icon class only when the mouse hovers over it.

p:not(.uninvited) Selects all p tags that DO NOT have the class .uninvited.

Scope

CSS variables are inheritable, so if you set the custom property on a parent element, it will be inherited by the child as well. But much like scope of a Javascript function, where variables inside a function cannot be accessed outside of it, no elements outside of the parent can access that variable. That's why most of the time, CSS variables are declared in the html element, because every element is a child of the html. It's a way of making the variables have a global scope.

If you don't want your custom properties to have a global scope, you can declare them in any specific CSS element.

Practical Uses

There are many more uses for CSS variables than simply decreasing redundancy. They can be used with Javascript to create interactive elements with reactive layouts, as well as in media queries and more.
